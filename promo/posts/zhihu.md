## 文章标题

```
用 Claude Code 找不到之前的会话？我写了个全文搜索工具
```

## 正文

```
天天用 Claude Code 的人应该都遇到过：`claude --resume` 只给你一列 AI 生成的标题。
标题没认出来就只能一路翻，可你分明记得上周调过那个偶发失败的测试、设计过那张表——
就是找不到是哪个会话。

问题的本质是：它让你「按标题找」，而你记住的是「当时干了什么」。

于是我写了 ccsearch（命令 ccs）：对整个 Claude Code 历史做全文、BM25 排序的搜索。
它索引的是每个会话的全文——你的提问、Claude 的回复、跑过的命令、动过的文件，甚至
subagent 的记录——而不只是标题。输入你还记得的词，选一个结果，终端就直接回到那个
项目目录并恢复会话。

几个我比较满意的点：

1）中文能搜。FTS5 自带分词做不了中文子串检索，所以我自己写了分析器：中文按重叠
二元组切（全文检索 → 全文 文检 检索），两个字的查询也能命中。

2）快且省心。单个 Rust 静态二进制，SQLite/FTS5 直接编进去，无运行时依赖，热搜索
<50ms。

3）隐私。100% 本地，会话不出本机；只有你主动用语义召回（-e）时才会把「查询词」
发出去。

开源在这（MIT）：https://github.com/yaalsn/ccsearch
如果你也是重度用户，欢迎试试看它能不能把你要找的那个会话顶上来。
```
